# 3D 智能商城（AI-Driven 3D Smart Mall）

## 项目简介

本项目是一个 **AI 驱动的三维智能导购商城系统**，前端基于 **Vue 3 + TypeScript + Three.js** 构建，通过引入 **AI Agent（智能代理）**，实现 **自然语言驱动的三维空间导航与交互**。

系统将传统二维导购模式升级为 **可交互的三维语义空间**，用户无需理解复杂界面或操作逻辑，仅通过文本/语音指令，即可完成店铺定位、路径导航、区域高亮等导购行为。

该项目定位为 **工程化、可扩展、可配置的 3D 导购系统原型**，适用于毕业设计、智能交互系统研究及三维 Web 应用探索。

---

## 核心目标

- 构建 **结构化、可维护的 Three.js 三维商城架构**
- 实现 **AI Agent 与 3D 场景的解耦式协作**
- 支持 **自然语言 → 行为指令 → 三维控制** 的完整闭环
- 保证商城结构 **不写死**，支持配置化与数据驱动

---

## 核心特性

- ✅ **三维商城空间建模（Three.js）**  
  商城以三维空间形式呈现，支持楼层、区域、店铺、货架等语义对象。

- ✅ **AI Agent 决策驱动**  
  AI 仅负责意图理解与动作决策，不直接操作 Three.js API。

- ✅ **标准化 Action 协议**  
  AI 输出结构化动作指令，由前端控制层统一执行，降低耦合度。

- ✅ **工程化前端架构**  
  采用现代前端工程规范（Vue3 + TS + Vite + Pinia），具备清晰分层。

- ✅ **非技术用户友好**  
  商城结构与语义信息通过配置/数据描述，而非硬编码在渲染逻辑中。

## 系统整体思路（概念层）

用户自然语言输入
↓
AI Agent（意图理解 / 决策）
↓
Action 指令（标准化协议）
↓
Three.js 控制层
↓
三维商城场景反馈


> **AI 不渲染场景，AI 只下达指令**  
> **Three.js 不理解语义，只执行行为**

---

## 系统整体执行流程（工程视角）

在前端工程中，自然语言交互并不会直接驱动 Three.js，
而是通过一条可校验、可追踪、可中断的执行路径完成。

标准执行流程如下：

用户输入（文本 / 语音）
↓
AI Agent（意图理解，生成 Action 建议）
↓
前端控制层（权限 / 状态 / 上下文校验）
↓
领域行为调用（语义级操作）
↓
Three.js 渲染引擎执行
↓
三维场景状态反馈（视觉 / 交互）

---

## 系统角色与运行模式速览

系统支持多角色在不同模式下运行：

- **运行态（Runtime Mode）**
  - 面向普通用户
  - 用于商城游览、导航与导购

- **配置态（Config Mode）**
  - 面向管理员 / 商家
  - 用于商城结构与店铺内容配置

不同角色在不同模式下可执行的行为由前端权限系统严格控制，
即使通过 AI 触发指令，也无法越权执行。
